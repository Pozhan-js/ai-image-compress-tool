# 图片压缩工具实现步骤总览

## 1. 范围与目标对齐
- 明确必须先交付的核心链路：上传 -> 参数配置 -> 压缩执行 -> 对比预览 -> 导出。
- 固定首版限制：单图 <= 50MB、单批 <= 100 张、默认本地处理。
- 输出统一的错误码与提示文案（格式不支持、体积超限、数量超限、压缩失败）。

## 2. 工程初始化与基础能力
- 按技术方案建立目录：`apps/web-client`、`apps/desktop-app`、`packages/api-server`。
- 配置开发脚本：并行启动 Web、API、Electron。
- 建立共享类型：任务状态、压缩参数、文件元数据、统计信息。
- 落地日志与全局错误处理（前后端都可追踪问题）。

## 3. 核心业务闭环（MVP）
- 完成上传与预览。
- 完成压缩参数与预设。
- 完成压缩任务创建、进度轮询、结果展示。
- 完成压缩前后对比（分屏/滑条至少一种）。
- 完成单张导出与批量导出。

## 4. 批量与稳定性增强
- 接入 worker 并发池（或队列）实现批处理与限流。
- 增加失败重试与部分失败可恢复机制。
- 增加历史记录与输出路径管理。
- 增加临时文件清理策略与磁盘阈值保护。

## 5. 体验与发布
- 增加主题切换、快捷键、空状态与引导文案。
- 补齐测试：单元、集成、端到端关键流程。
- 桌面打包与多平台构建（macOS 优先）。
- 发布前做性能验收：启动时间、单图耗时、批处理内存占用。

## 里程碑建议（对应技术方案）
- M1（第 1-2 周）：上传 + 基础压缩 API + 前端参数面板。
- M2（第 3-4 周）：批量任务 + 进度 + 预设模式。
- M3（第 5-6 周）：对比视图 + 历史记录 + 导出。
- M4（第 7-8 周）：测试、调优、桌面打包发布。
